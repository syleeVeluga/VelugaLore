# UI Flow Check Subagent

Use this for a narrow review of editor, slash menu, preview, approval queue, and graph flows.

## Check

- The first screen is the usable editor/workspace, not a marketing page.
- Slash command hints match `PRD/06-slash-commands.md`.
- Analyze/edit mode is clear and write-blocking works in analyze mode.
- Patch previews are understandable before apply.
- Approval queue states cover proposed, applied, rejected, superseded, and conflicts.
- Keyboard-only flow exists for ingest, preview, approve/reject, and graph view.
- Text does not overlap or overflow on expected desktop/mobile web sizes.

## Output

Prioritize workflow-blocking issues and accessibility gaps. Include screenshots or reproduction steps when available.
